# Season Product Analyst

Use this skill to write the Markdown report that follows `scripts/season_analysis.sh`.

This skill is specific to the seasonal Shopify reporting pipeline in this repository. It should be used for:

- single-season reports such as `winter25`
- same-season cross-year comparisons such as `winter25` vs `winter26`
- product-image-led narrative analysis grounded in the season JSON outputs
- official product-detail interpretation from Admin GraphQL descriptions, tags, selected options, and metafields

## Source Of Truth

Read the latest season-analysis output folder produced by the workflow.

- Single season: `<brand>_<season>_report.json`
- Comparison: `<brand>_<family>_comparison_report.json` plus the season JSON files in `report_source/`

Use the JSON fields that actually exist in the run:

- `season`
- `report_period`
- `season_product_performance.rows`
- `season_product_performance.top_20_rows`
- `season_product_performance.bottom_20_rows`
- `season_product_performance.product_detail_enrichment_summary`
- `product_detail_enrichment_summary`
- `product_image_focus.top_10_products`
- `product_image_focus.bottom_10_products`
- `product_image_index`
- `rows[].product_detail`
- `rows[].product_detail.official_product_attributes`
- `rows[].product_detail.metafields_normalized`
- `comparison`
- `comparison_summary`
- `seasons.*`

Do not invent missing metrics, product attributes, fabric composition, image matches, or visual observations.

## Report Rules

- Write one Markdown report only.
- Treat ShopifyQL metrics as commercial truth for sales, returns, discounts, and units.
- Treat Admin GraphQL product descriptions, tags, selected options, and metafields as official product-detail truth when available.
- Treat local product images as visual/design evidence and use them as a separate qualitative layer, not merely as a fallback.
- Keep row order and ranking integrity intact.
- If the input is a comparison report, compare the same season across years rather than treating the seasons as different families.
- Do not mention absolute local filesystem paths in the report body.
- Do not use CDN image links in the report body.
- Use generic provenance language such as:
  - “This report is based on the JSON generated by the season analysis workflow.”
  - “Product images were retrieved through Shopify GraphQL and stored locally for analysis.”
- If image status is `ambiguous`, `not_found`, `skipped`, or missing `local_path`, do not make visual claims for that product.
- If exact fabric composition is missing, write `Official composition: Unknown`.
- Exact fabric composition and fibre percentages must only come from official product data. Never infer them from images.
- If image-based fabric interpretation is used, label it `Visual inference`.
- Return-related design causes are hypotheses unless directly supported by customer return reasons.

## Image Guidance

- Use `product_image.local_path` only.
- Prefer compact Markdown or HTML figure blocks with captions when an image directly supports the insight.
- Keep figures small to medium sized and place them next to the related observation.
- Use images to evaluate silhouette, proportion, length, fit impression, styling, print scale, colour impression, fabric surface appearance, drape, structure, opacity, texture, use case, and visual commercial interpretation.
- If a silhouette, fabric, or construction read is a visual inference, label it explicitly as `Visual inference`.
- If an image is unavailable, say image unavailable. If an image is unclear, say image unclear.

## Official Product Detail Guidance

- Use `product_detail.official_product_attributes` for official material/fabric, colour, fit, neckline, sleeve length, clothing features, product features, product size, origin country, and collection name.
- Use `metafields_normalized.materials.text`, `product_size.text`, and `product_features.text` as parsed official rich-text fields.
- Use resolved `shopify.*` metafield labels, not raw GIDs, when available.
- Selected options can support official colour/size reads for the matching variant.
- If official product detail enrichment is unavailable, say so in the data confidence note and continue with ShopifyQL plus image evidence.

## Analysis Guidance

- For single-season reports, analyze the full season rows and explain the commercial story, product winners and losers, silhouette patterns, colour direction, print direction, fabric/material evidence, use case, and returns-as-design-feedback.
- For comparison reports, highlight what repeated, improved, weakened, or disappeared across the two seasons across commercial, official product-detail, and visual layers.
- Use helper slices like top/bottom rows to support the narrative, not as the only evidence.
- Distinguish product-level findings from season-level or collection-level trends.
- Ground recommendations in observed report patterns and practical actions for design, merchandising, and marketing.
- Prefer compact product cards, grids, boards, and evidence tables over long text-only sections.

## Expected Structure

Single-season reports should use:

- Executive visual summary
- Methodology and data confidence note
- Season performance snapshot
- Product performance cards
- Hero product grid
- Underperformer/risk product grid
- Silhouette grid
- Colour board
- Print board
- Fabric/material evidence table
- Use-case map
- Returns-as-design-feedback table
- Core repeatable ideas
- Product gaps
- Recommendations for next collection

Same-family comparison reports should use:

- Side-by-side season summary cards
- Methodology and data confidence note
- Category mix comparison
- Hero product comparison grid
- Repeated-style case studies if repeated styles exist
- Silhouette comparison grid
- Colour arc comparison
- Print engine comparison
- Fabric/material evidence comparison
- Use-case comparison
- Returns comparison
- Diagnostic matrix
- Next collection playbook

## Product Card Guidance

Product cards should include, where available:

- product image
- product title
- net sales
- units sold
- return rate
- discount rate
- official material/fabric
- visual design read
- verdict

Allowed verdicts include: Protect, Repeat, Refine, Redesign, Reduce, Avoid, Watch, Design-led but commercially weak, Strong sales but high returns, Good idea, weak execution, Commercially strong, fit-risk, Visual hero, margin-risk.

## Output

- Markdown only.
- No preamble.
- No process notes.
- No tool logs.
